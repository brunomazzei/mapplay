O MAPPLAY é uma plataforma digital full stack (web e mobile) fundamentada nos conceitos de Crowdsourcing e Informação Geográfica Voluntária (VGI). O sistema atua como uma rede social esportiva, transformando o cidadão em um sensor social para mapear espaços públicos destinados à prática de basquete, futebol e skate. A plataforma integra ferramentas de zeladoria comunitária e gestão de eventos, visando reduzir a "opacidade do território" e promover a ocupação democrática do espaço urbano.

2. Regras de Negócio (RN)
As diretrizes operacionais do sistema seguem o modelo de consenso e transparência:
RN01 - Cadastro por Consenso Democrático: A validade das informações segue o princípio da inteligência coletiva. Novos espaços públicos só serão registrados no mapa quando pelo menos 3 usuários registrarem a mesma localização no aplicativo, garantindo a precisão dos dados sem a necessidade de moderadores centrais.
RN02 - Exclusividade de Espaços Públicos: O cadastro é estritamente restrito a equipamentos do Sistema de Lazer Esportivo público, como praças e centros comunitários.
RN03 - Geolocalização Obrigatória: É indispensável o envio de coordenadas geográficas exatas (latitude/longitude) para eliminar imprecisões e garantir uma base cartográfica fidedigna.
RN04 - Atributos Técnicos por Modalidade: O formulário de cadastro exige dados específicos: altura da cesta para basquete, tipo de obstáculo para skate e tipo de gramado para futebol.
RN05 - Limitação de Responsabilidade na Zeladoria: A plataforma facilita a organização comunitária, mas não se responsabiliza pela manutenção física dos locais; os usuários organizam mutirões por iniciativa própria através do fórum.
RN06 - Acesso aos Dados e Transparência: Estatísticas básicas são abertas à comunidade, enquanto dashboards analíticos são voltados para perfis administrativos e parceiros governamentais.

3. Funcionalidades Principais
3.1 Estrutura de Perfis Sociais
Perfil do Usuário: Cada usuário possui um perfil público contendo nome, região, esportes praticados e um registro histórico de atividades, incluindo eventos criados, participações em "peladas" e engajamento em revitalizações.
Perfil dos Espaços: Semelhante ao Google Maps, apresenta nome, endereço, fotos, avaliações e registro de atividades. É o ponto central para a criação de eventos e mobilizações específicas daquele local.
3.2 Mapeamento e Diagnóstico Colaborativo
Consulta Centralizada: Visualização georreferenciada com filtros por modalidade, bairro, iluminação e pavimentação.
Sistema de Avaliação em Três Níveis: As condições dos espaços são classificadas obrigatoriamente como: "Ótimo", "Necessita de reparos" ou "Somente com revitalização".
Atualização do status do espaço: Usuários podem enviar fotos recentes para comprovar o estado do local.
3.3 Mobilização e Engajamento Comunitário
Prática Esportiva: Na página de cada espaço, usuários podem criar eventos detalhando jogos ("peladas" ou "rolês") para que outros interessados tenham visibilidade e participem.
Zeladoria Coletiva: É possível criar eventos de mobilização para revitalização. O sistema pode incluir notificações para usuários nas proximidades para incentivar a participação nesses mutirões.
